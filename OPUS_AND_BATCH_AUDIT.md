# Claude Opus 4.5 & 分批审稿功能说明

## 新增功能

### 1. Claude Opus 4.5 模型支持

已添加 Claude Opus 4.5 模型，这是目前最强大的 Claude 模型，具有以下特点：

- **超长上下文**: 支持 200K tokens（约 60 万字中文）
- **最强推理能力**: 适合复杂的剧本分析和评估
- **更长超时时间**: 针对超长剧本，Opus 模型的超时时间最长可达 30 分钟

#### 配置信息
```typescript

Base URL: https://mixai.cc/v1
```

#### 超时配置（Opus 专属）

| 剧本长度 | Opus 超时 | Sonnet 超时 | 其他模型超时 |
|---------|----------|------------|-------------|
| < 5万字 | 5 分钟 | 5 分钟 | 3 分钟 |

| > 15万字 | **30 分钟** | 20 分钟 | 15 分钟 |

### 2. 分批审稿功能（支持 20 万字以上剧本）

针对超长剧本（>20 万字），系统会自动启用分批审稿模式。

#### 工作原理

1. **自动检测**: 当剧本长度超过 20 万字时，自动切换到分批模式
2. **智能分批**: 每批次处理 8 万字，确保 AI 能够深入分析
3. **进度显示**: 实时显示当前处理进度（第 X/Y 批次）
4. **结果合并**: 自动合并所有批次的审稿结果

#### 分批策略

```typescript
CHUNK_SIZE = 80000 字/批次

示例：
- 20 万字剧本 → 3 批次
- 30 万字剧本 → 4 批次
- 50 万字剧本 → 7 批次
```

#### 结果合并逻辑

- **批注**: 保留所有批次的批注，标注批次来源
- **维度分数**: 计算所有批次的平均分数
- **总分**: 所有批次总分的平均值
- **摘要**: 自动生成分批审稿报告

#### 使用方法

**自动模式**（推荐）:
- 直接上传超长剧本
- 系统自动检测并启用分批模式
- 无需手动配置

**手动选择 Opus 模型**:
1. 在审稿界面右上角选择 "Claude Opus 4.5 (MixAI)"
2. 点击"开始审稿"
3. 系统会根据剧本长度自动决定是否分批

### 3. 进度条显示

分批审稿时，界面会显示：
- 当前批次 / 总批次数
- 当前处理的字数范围
- 进度条（百分比）

示例：
```
分批审稿中 (2/4)
正在审稿第 2/4 批次 (80000-160000字)
[████████████░░░░░░░░] 50%
```

## 使用场景

### 场景 1: 全本剧本审稿（20 万字以上）

**需求**: 对一部 30 万字的完整剧本进行深度审稿

**操作**:
1. 选择 "Claude Opus 4.5 (MixAI)" 模型
2. 上传完整剧本
3. 点击"开始审稿"
4. 等待分批处理完成（约 15-20 分钟）

**结果**: 获得完整的审稿报告，包含所有批次的问题汇总

### 场景 2: 小说原文 vs 改编剧本对比

**需求**: 评估改编是否恰当（节奏压缩、主线一致性等）

**操作**:
1. 准备两个文件：
   - 小说原文（可能 50 万字）
   - 改编后的剧本（可能 20 万字）
2. 使用 Opus 模型进行对比分析
3. 系统会自动分批处理

**提示**: 可以在 Prompt 中明确要求 AI 关注：
- 节奏压缩是否合理
- 主线情节是否一致
- 关键情节点是否保留
- 人物弧光是否完整

### 场景 3: 超长连续剧审稿

**需求**: 对 100 集短剧（总计 50 万字）进行全本审稿

**操作**:
1. 合并所有集数为一个文件
2. 选择 Opus 模型
3. 启用分批审稿（自动）
4. 等待约 30-40 分钟

**优势**:
- 能够发现跨集的连贯性问题
- 检测人物设定的前后矛盾
- 评估整体节奏和情绪曲线


### 错误处理

- 单个批次失败不会中断整体流程
- 继续处理剩余批次
- 最终报告中会标注失败的批次

### 性能优化

- 批次之间串行处理（避免 API 限流）
- 每批次独立超时控制
- 自动重试机制（可选）

## 注意事项

### 1. 时间成本

- 20 万字剧本（3 批次）: 约 15-20 分钟
- 50 万字剧本（7 批次）: 约 40-50 分钟
- 建议在非高峰时段运行

### 2. API 成本

- Opus 模型成本较高
- 分批处理会增加 API 调用次数
- 建议先用 Sonnet 测试，确认无误后再用 Opus

### 3. 批次边界问题

- 分批可能会在场景中间切分
- 某些跨场景的问题可能被遗漏
- 建议结合人工复查

### 4. 内存占用

- 超长剧本会占用较多浏览器内存
- 建议关闭其他标签页
- 如遇卡顿，可刷新页面重试

## 故障排查

### 问题 1: 分批审稿中断

**原因**: 网络不稳定或 API 超时

**解决**:
1. 检查网络连接
2. 增加超时时间
3. 减小批次大小（CHUNK_SIZE）

### 问题 2: 内存溢出

**原因**: 剧本过长，浏览器内存不足

**解决**:
1. 关闭其他标签页
2. 使用 Chrome 浏览器（内存管理更好）
3. 考虑分多次审稿

### 问题 3: 结果不完整

**原因**: 某些批次失败

**解决**:
1. 查看控制台日志，找到失败的批次
2. 单独审稿失败的部分
3. 手动合并结果
